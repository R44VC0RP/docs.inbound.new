---
title: "Get Scheduled Email"
api: "GET /api/v2/emails/schedule/{id}"
description: "Retrieve details of a specific scheduled email by ID"
---

<Note>
This endpoint provides complete details of a scheduled email, including its current status, delivery attempts, and all original content.
</Note>

## Authentication

This endpoint requires authentication via:
- **API key auth**: Bearer token in Authorization header

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier of the scheduled email to retrieve.
</ParamField>

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication.
</ParamField>

## Request Example

<RequestExample>
  ```javascript Inbound SDK
  import { Inbound } from '@inboundemail/sdk';

  const inbound = new Inbound('your_api_key');

  // Get scheduled email details
  const { data, error } = await inbound.email.sent.getScheduled('sch_1234567890abcdef');

  if (error) {
    console.error('Error:', error);
  } else {
    console.log('Email status:', data.status);
    console.log('Scheduled for:', data.scheduled_at);
    console.log('Subject:', data.subject);
    
    if (data.status === 'failed') {
      console.log('Last error:', data.last_error);
    }
  }
  ```

  ```bash cURL
  curl -X GET 'https://inbound.new/api/v2/emails/schedule/sch_1234567890abcdef' \
       -H 'Authorization: Bearer your_api_key'
  ```

  ```python Python

  response = requests.get(

  print(response.json())
  ```

<ResponseField name="scheduled_at" type="string" required>
  ISO 8601 timestamp of when the email is/was scheduled to be sent.
</ResponseField>

<ResponseField name="timezone" type="string" required>

</ResponseField>

<ResponseField name="status" type="string" required>

</ResponseField>

<ResponseField name="attempts" type="integer" required>
  Number of delivery attempts made.
</ResponseField>

<ResponseField name="max_attempts" type="integer" required>
  Maximum number of delivery attempts allowed before marking as failed.
</ResponseField>

<ResponseField name="next_retry_at" type="string">
  ISO 8601 timestamp of when the next delivery attempt will be made (only present if status is 'failed' and more attempts remain).
</ResponseField>

<ResponseField name="last_error" type="string">
  Error message from the last failed delivery attempt (only present if status is 'failed').
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the email was originally scheduled.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of when the email record was last updated.
</ResponseField>

<ResponseField name="sent_at" type="string">
  ISO 8601 timestamp of when the email was successfully sent (only present if status is 'sent').
</ResponseField>

<ResponseField name="sent_email_id" type="string">
  ID of the sent email record (only present if status is 'sent').
</ResponseField>

## Status Information

<AccordionGroup>
  <Accordion title="Status Values">
    - **`scheduled`**: Email is waiting to be sent at the scheduled time
    - **`sent`**: Email was successfully delivered
    - **`failed`**: Email failed to send after all retry attempts
    - **`cancelled`**: Email was cancelled before being sent
  </Accordion>
  
  <Accordion title="Retry Logic">
    Failed emails are automatically retried up to `max_attempts` times with exponential backoff:
    - Attempt 1: Immediate
    - Attempt 2: After 1 minute
    - Attempt 3: After 5 minutes
    
    The `next_retry_at` field shows when the next attempt will occur.
  </Accordion>
  
  <Accordion title="Error Messages">
    Common error messages in `last_error` field:
    - "Recipient email address rejected by server"
    - "Domain not found"
    - "Rate limit exceeded"
    - "Attachment processing failed"
  </Accordion>
</AccordionGroup>
