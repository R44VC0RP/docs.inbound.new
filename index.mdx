# Inbound API v2

Transform any email address into a powerful API endpoint. Inbound provides a simple yet robust way to receive, process, and manage emails programmatically.

## Base URL

All API requests are made to:

```
https://inbound.new/api/v2
```

## Getting Started

<Steps>
  <Step title="Get Your API Key">
    Sign up at [inbound.new](https://inbound.new) and generate an API key from your dashboard.
  </Step>
  
  <Step title="Set Up Authentication">
    Include your API key in the Authorization header of all requests:
    
    ```bash
    Authorization: Bearer your_api_key_here
    ```
  </Step>
  
  <Step title="Make Your First Request">
    List your emails to verify authentication:
    
    ```bash
    curl -X GET https://inbound.new/api/v2/mail \
      -H "Authorization: Bearer your_api_key_here"
    ```
  </Step>
</Steps>

## Quick Example

Here's a complete example of retrieving and processing emails:

<CodeGroup>
```bash cURL
curl -X GET https://inbound.new/api/v2/mail \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://inbound.new/api/v2/mail', {
  headers: {
    'Authorization': `Bearer ${process.env.INBOUND_API_KEY}`,
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data.emails);
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://inbound.new/api/v2/mail',
    headers=headers
)

data = response.json()
print(data['emails'])
```
</CodeGroup>

## Response Format

All API responses follow this consistent format:

```json
{
  "emails": [...],
  "pagination": {
    "total": 150,
    "limit": 50,
    "offset": 0,
    "hasMore": true
  },
  "filters": {
    "uniqueDomains": ["example.com", "test.com"]
  }
}
```

## Error Handling

API errors are returned with appropriate HTTP status codes and descriptive error messages:

```json
{
  "error": "Invalid or missing API key"
}
```

Common status codes:
- `200` - Success
- `400` - Bad Request (invalid parameters)
- `401` - Unauthorized (invalid API key)
- `404` - Not Found (email not found)
- `500` - Internal Server Error
